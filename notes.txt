    -> NOTE: HTTP (HyperText Transfer Protocol) and HTTPS (Secure) are protocols 
    for "transmitting web data". HTTP sends data in plain text, making it insecure, 
    while HTTPS uses SSL/TLS encryption to protect data integrity and user privacy.
    HTTP port is 80 & HTTPS port is 443


    -> NOTE: SSH The Secure Shell (SSH) protocol is a cryptographic network protocol used 
    for secure, encrypted communication between a client and a server over an 
    insecure network. It enables "remote login" using SSH Key Pairs (Public & Private Keys)
    Above is mandatory for Linux Servers 
    SSH port is 22


    -> NOTE: Simple Network Management Protocol (SNMP) is an 
    application-layer standard protocol for "monitoring" and managing 
    network-connected devices (routers, switches, servers,firewalls, IoT) in IP networks. 
    It operates over UDP (ports 161/162) to track network performance, 
    detect faults, and configure remote devices.

    -> Modules / Key Concepts Of ITOM

        -> CMDB (Configuration Management Database)

            -> The "backbone" of ITOM. 
            All discovered assets and their relationships are 
            stored here as Configuration Items (CIs).

        -> Configuration Items (CI)

            -> Any component that needs to be managed — servers, databases, 
            applications, network devices, etc.
        
        -> Discovery 

            -> The process of automatically "identifying" and "populating" CIs 
            into the CMDB. Uses MID Server to communicate with the network.

        -> MID Server (Management, Instrumentation & Discovery)

            -> A Java-based application(Agent) installed on a Windows/Linux machine 
            within your network(cloud/on-premise). 
            Acts as a bridge between ServiceNow and the target infrastructure.

        -> Service Mapping 

            -> Creates a Map linking a business service (e.g., "Email Service") 
            to all its dependent infrastructure components.

        -> Event Management 

            -> Collects Alerts from External Monitoring Tools(Nagios, Prometheus, Cloudwatch etc)
            and correlates them into actionable alerts.
